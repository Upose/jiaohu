Eagles之热力图应用

一、热力图背景

       所谓热力图，即以特殊高亮的形式显示访客热衷的页面区域和访客所在的地理区域的图示。通过设置热力图半径、颜色、透明度等参数直观展示数据分布情况。
       热力图实际上也是基于出现频率的一种数据处理方法。网站热力图，即是对页面点击频率的一种数据搜集；地图热力图，则是对已存在坐标在地图数据上的重叠显示。
       这里着重介绍下Eagles在大数据背景下对地图热力图的处理能力。

二、处理方法

       2.1Eagles热力图处理的优势

           Eagles提供了丰富的聚合/分类算法，利用其冗长但是强大的Aggregation DSL可以表达出比SQL还要复杂的聚合逻辑， 为数据分析提供了有力的支撑， 目前Eagles支持：
           域的折叠与融合；
           百分位等级聚合，展示了观测值在某个特定值之下的百分率；
           地理范围聚合，提供了一个覆盖了所有位置值的范围框图。

       2.2数据处理流程
       第一步：
               从大量数据中获取有效地址数据，和地图搜索类似的，热力图同样是基于地理位置坐标的一种数据处理方法。
               数据获得途径可以很多，应用场景也很多。例如，App可以从用户手机上获取定位权限，从而获取地理位置相关数据。
               这里主要结合Eagles的具体功能来了解Eagles的数据处理能力。
               无论如何，获取的数据中至少包含以下几个字段：
               location 类型为地理坐标类型，用于标志地理位置
               content 类型为字符串，可以用于区分location的具体意义
               time类型为时间，对时效性有特殊要求时可存在

      上图是一个简单示例，数据包含了content,location,time字段。
       第二步：
                针对不同的搜索要求给出不同的查询语句。
                例如：想要形成违规停车重灾区的热力图
                那必须要获得相关的违停数据，这组数据需要包括时间字段(time、date)， 地理坐标（location）、描述（content）,在Eagles处理后形成不同时间的热力图。之后，针对每一组数据，对数据的location字段进行统计，查询出每个地址坐标对应的条数，count出通过的数量。
                这里还是可以通过范围聚合来实现，主要方法是根据已知点的坐标，对它进行范围聚合。范围聚合的原理就是对一个已知点，对一定范围（精度可以设置为由小渐大）来进行数据采样，根据范围采样得出的hit结果来判断出附近的count值。
                例如对于某个地标建筑，或某些大型超市群，这里可能是违停情况的的重灾区，那么就可以对这些点进行范围查询。

 {
    "query":{
"filtered":{
    "query":{
        "match":{
            "content":{
                "query":"illegal parking",
                "operator":"and"
            }
        }
    },
"filter":{
    "geo_distance":{
            "distance":5,
            "unit":"m",
            "location":{
                "lat":116.418261,
                "lon":39.921984
                      }
                   }
              }
         }
    }
}

       范围查询实际上是对范围外点的过滤，之返回范围内的结果。这里的违停重灾区地理坐标为（116.418261,39.921984），同时需要描述content为illegal parking。 对distance设置不同的大小可以得到不同的精度，具体情况根据实际情况来设定。

（图中搜索出5个符合条件的点）

      除了范围查询的方法，Eagles还提供了geohash来对地理坐标进行处理。GeoHash将二维的经纬度转换成字符串，每一个字符串代表了某一矩形区域。也就是说，这个矩形区域内所有的点（经纬度坐标）都共享相同的GeoHash字符串，这样既可以保护隐私，又比较容易做缓存。这样，当字符串的长度足够大时，精度也会足够高。

{
"aggs":{
    "heat_map":{
        "geohash_grid":{
            "field":"location",
            "precision":9
         }
     }
    }
}

      precision可对字符串位数的设置，geohash的编码方式是将地理坐标进行二进编码为字符串，字符串位数越多，系统消耗越大，精度也越高，9位可精确到0.5m。
      还可利用Eagles的facets聚合，通过计算geo_distance切面，能够获得落入到从给定位置到某距离范围内的文档的信息。

{
"facets":{
    "heat_map":{
        "geo_distance":{
            "field":"location",
            "unit":"m",
            "location":{
                "lat":116.418261,
                "lon":39.921984
            },
           "ranges":[
               {"to":10},
               {"from":10,"to":100}
                    ]
          }
      }   
    }
}

       第三步：
            查询结果得出后，可对特定坐标或一定范围内坐标进行进行count统计，根据结果的密集程度从而可绘制热力图。
            当然，以上只是对一个点的地理坐标进行了查询。实际中，根据不同的需求，Eagles会对大量有意义的数据进行查询。所以，通常，在整合了分析数据后会得到这样的一组数据。
聚合结果会输出对应点地理坐标（lng,lat）,出现此处的数量（频度）count。 根据这组数据，之后可利用canvas等等来绘制热力图。

       2.3分析总结

           对热力图的分析实质上也是对地图搜索的一种情况，都是依靠地理坐标进行的搜索统计。利用Eagles对含有地理坐标的数据做出不同的处理就能为不同的目的分析出数据。
