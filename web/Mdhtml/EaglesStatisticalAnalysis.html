Eagles数据统计分析

一、背景介绍
         所谓统计分析，就统计学来说，是指运用统计方法及与分析对象有关的知识，从定量与定性的结合上进行的研究活动。
        通俗点来说，就是能帮用户从大量数据中总结事物的现在情况，预测它将来走势。这一点和Eagles提供的功能不谋而合。Eagles提供实时搜索和准确分析，对数据进行快速准确的统计分析，可以给用户的数据带来更深层的价值
        统计分析带来的意义不言而喻，各行各业都是需要利用这点合理调整工作规划。小到商城对销售数据的统计分析，大到国家对民生数据的统计分析。对于商城而言，统计分析的目的不外乎通过对销售数据的即使分析，来辅助其他部门（采购、运营）及时调整策略。能想到的最简单方面就是关键词+数字数据的综合分析（即商品+销量+利润）。Eagles查询可以对数据进行简单/复杂的词组匹配，同时还可以对数字数据的统计（包括数字字段的最小值、最大值、和值、平均值、文档数）。例如:
        其实，这些方面都被Eagles的聚类分析功能所囊括。
        聚合分析主要是对文档进行数据分析，获得一些统计型的数据，例如词频，均值等，从而从大量数据中分析出有价值的结果，从而辅助用户依照趋势调整策略。

二．具体实施
       2.1聚合分析方法
           聚合分析分为很多种类型，每种类型都有自己的目的和输出。在查询之前，需要提到aggregations对象（可简写为aggs），聚类分析主要都是在aggs语句下进行的查询。

           基本的结构为：

{
"aggs" : {
       具体参数的语句
        }
}

           具体参数中，根据应用场景的不同，可以添加不同的参数。可能包括以下几种：
           1、数学统计值。min、max、sum、count、avg，sumofsquares，variance，stddeviation、stddeviationbounds
           2、过滤器（filter）聚合。通过嵌套过滤器来提高统计精度。
           3、缺失（missing）聚合。查找指定文档中没有某个字段的文档数
           4、词（terms）聚合 。返回指定字段的每种值所占的个数,词聚合下还可以包括size，order，mindoc_count，include，exclude.
           5、范围（range）聚合,包括数字聚合，日期范围聚合，IP地址聚合
           6、直方图(histogram)聚合。直方图根据给定的间隔，返回一系列相同间隔的结果。

           以下结合舆情分析案例简单分析下该模块的具体作用：
           舆情分析根据特定问题的需要，对针对这个问题的舆情进行深层次的思维加工和分析研究，得到相关结论的过程。
           实际上，它的统计分析，无非还是需要3个过程:数据收集，数据统计，数据分析。


       2.2数据搜集
           这方面Eagles的便利之处不言而喻，Eagles提供了对多种数据源的支持，通过整合Crab数据收集引擎，能够支持多种数据源的定时收集，如传统ETL工具，网页Spider， 数据库，文件系统，邮件，RabbitMQ消息队列, Log 等数据源，与此同时，还可完全自定义索引结构.
           具体方法可以是网络爬虫从各大主流门户网站抓取相关信息，或是从政府信息反馈网、投诉平台（论坛）获得反馈数据等等。
       2.3基于数学统计值的统计分析
           超市是个对数字比较敏感的场景，Eagles能通过对数字进行有效分析来找出顾客消费特点，从而帮助超市调整运营策略。鉴于此，这儿先利用Eagles来对超市后台的销售数据进行些处理分析
           对统计数据值，可能包括以下字段和模块
           会员 IDnumber（字符）
           促销占比 PromotionC（双精度浮点）
           交易流水号 SerialNumber(字符)
           结账时间 Time（时间）
           消费总额 TotalCost（双精度浮点）
           商品名 TradeName（字符串）
           会员 VIP（Bool）
           性别 gender（字符串）
           并存有以下数据
           对于超市而言，进行数据采集不是徒劳，通过Eagles的数据分析可以从大量数据中提取出顾客的消费习惯，如果超市迎合了顾客的消费习惯，那显然会提高自己的经济效益。
           a)常用数字统计值的分析
        常用统计值，即求指定数字字段的最小值、最大值、和值、平均值、文档数。

{
"aggs":{
    "min_cost":{
        "min":{
            "field":"TotalCost"
        }
    },
    "max_cost":{
        "max":{
            "field":"TotalCost"
        }
    },
    "sum_cost":{
        "sum":{
            "field":"TotalCost"
        }
    },
    "avg_cost":{
        "avg":{
            "field":"TotalCost"
        }
    },
    "count_cost":{
        "value_count":{
            "field":"TotalCost"
         }
    }
  }
}

        通过对数据的总体分析能得出总体的趋势,包括：最高消费、最低消费、平均消费、总销售额、成交笔数。

{

       2.4基于数据关键词的统计分析
            Eagles主要参与数据的统计分析，利用Eagles的查询功能来实现。
            这里还得提到Eagles对查询框架的支持。Eagles完整的支持了基于JSON的QueryDSL通用查询框架，QueryDSL是一个Java开源框架用于构建类型安全的SQL查询语句。它采用API代替拼凑字符串来构造查询语句。
            除了QueryDSL查询语法的支持，Eagles还支持类SQL的查询方式， 让熟悉数据库的你轻松上手，目前支持常用语法Select, Delete, Where, Order By, Group By, And/Or, Like,Count, Sum, Between等。
            除了超市这类对销售数字相当敏感的例子，对于政府或机构来说，市民的满意度是衡量自己是否合格的重要指标。那么，面对网络上的大量数据，如何利用Eagles来处理这些评价，帮助政府或机构采取及时的应对措施？
            就舆情统计而言，很多时候是基于关键词的一种分析，想要充分利用Eagles对数据的统计分析功能，应该至少包括以下部分：

       a)基于关键词的标签云


标签云实际上是对关键词词频统计的一种应用场景。
具体查询语句的关键词：
aggs：aggregations
terms：词聚合
field：设置为所有字段
include：即包括，正则表达式\u4E00-\u9FA5是汉字,修饰变量2代表只处理2个字以上的词
exclude:即不包括，可对指定冗余无意义的词进行过滤

{
"size": 0,
"aggs":{
    "tags":{
       "terms":{
            "field":"_all",
           "include":"[\u4E00-\u9FA5]{2,}",
"exclude":[ "好的"]
        }
    }
}
}


       在正确查询后，响应结果会给出对应词key，以及相应词的词频doc_count
       在提供的词频数据基础上，可以得到关键词的统计数据，则可实现类似舆情分析云标签。

       b)基于关键词的民情趋势

       民情趋势也是类似的，实质上还是进行词频的统计，只是对分析结果的不同的处理方式而已

       c)基于范围的正负面评价

{
"query" : {
        "terms" : {
          "_all" :[ "交管局", "交警"]
 }
},
    "aggs" : {
        "senti_score" : {
            "range" : {
                "field" : " senti_score ",
                "ranges" : [
                    { "to" : 0},
                    { "from" : 1}
                ]
            }
        }
    }
}

Eagles在词查询的基础上进行范围查询，然后对范围内数据count出结果。
range:范围，其下可对field字段，to/from具体范围进行细化
可分为{ "to" : X1 },
          { "from" :X1, "to" : X2},
          { "from" : X2 }

除此之外，根据实际应用场景的不同还可以对IPV4地址，时间等进行不同的范围聚合。
例如：
对时间范围聚合
          "ranges": { "to": "now-10M/M" },{ "from": "now-10M/M" }]
        第一个范围代表十个月前的月首之前。 第二个范围代表十个月前的月首之后。
对IP地址范围聚合:
           "ranges" : [{ "to" : "10.0.0.5" },{ "from" : "10.0.0.5" } ]
总之，Eagles提供了丰富的范围查询方法。

       d)基于关键词的驾校热力图

        和之前提到的热力图构建方法一样，需要对数据进行提取整合。 例如：想要通过Eagles对各驾校热点进行热力统计。
        那么，首先就会考虑到对数据进行提取，简单来说，热力图需要地理坐标+命中点数，所以实际中就是需要所有驾校地址+所有驾校报名人数。涉及到人数，那就想到aggs的聚合功能。

{
"aggs":{
    "findschool":{
        "terms":{
            "field":"driveschool"
            }

        }
    }
}

       e)基于时间段的热词统计

        数据分析往往具有时效性，对于实时的搜索引擎Eagles来说，对过时数据的把握能力很是重要。其实在实际运用中，Eagles对范围时间内的热词同样能做到精确的查询。
        这里需要运用到时间聚合，对于date字段，可以使用date_range对特定字段进行时间上的筛选。

{
"aggs":{
    "timedata":{
        "date_range":{
            "field":"time",
            "ranges":[
                {
                "from":"now-7d",
                "to":"now"
                }
                      ]
                }
          }
    }
}
